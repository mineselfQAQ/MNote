命名规范

无论在任何地方编写代码，命名规范是很重要的一点，这确保了项目代码的一致性
具体使用哪种命名规范不重要但是一定要确立一种规范

规范具体如下：　　protected等价于private，internal等价于public

  • 变量/常量
    • private：使用前缀_+Camel
    • public：使用Pascal
    • static：
      • private static：使用前缀s_+Camel
      • public static：使用Pascal
    • readonly：使用前缀_+Camel
    • static readonly
      • private static readonly：使用前缀s_+Camel
      • public static readonly：使用Pascel
    • const：使用全大写+_分割
  • 属性：使用Pascal
  • 枚举：使用Pascal
  • 事件：使用前缀On+Pascal
  • 类名/方法：使用Pascal 
    public class TClass 
{
    public void Foo1() {/*...*/}
    public static void Foo2() {/*...*/}
}
public static class StringExtension
{
    public static void Foo3(this string s) {/*...*/}
}

稍微总结一下就是：
一般都是用Pascal，对于private的变量，会使用前缀+Camel的命名方式
