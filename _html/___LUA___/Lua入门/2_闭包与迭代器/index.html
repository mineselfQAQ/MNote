前置基础

由于函数是第一类值，所以数字和字符串是具有相同权力的
这也意味着：函数可以存储到变量/table中，可以作为实参传入函数，也可以作为返回值

由于Lua函数可以嵌套，且内部函数可以访问外部变量，所以可以应用函数式编程
需要注意的是：函数是匿名的
一般情况下我们会这么写函数：
function foo(x) return 2*x end
上述写法其实是一种语法糖，事实上这等价于：
foo = function(x) return 2*x end
即foo是一个变量，存储了一个匿名函数
所以说：function() ... end其实是类似于{}(table构造式)的函数构造式

以另一个函数为参数的函数被称为高阶函数
可以有：table.sort(network, function(a,b) return (a.name > b.name) end) 此时sort函数高阶函数

在table中，由于函数是第一类值，我们可以将其定义为局部函数，有2种方法：

-- 方法1
local fact -- 前向声明
fact = function(n)
  if n == 0 then return 1
  else return n * fact(n-1) -- 虽然此时fact()还是没有定义完成，但是至少指代的是local的fact，解释型语言自然能知道fact是什么
  end
end

-- 方法2
local function fact(n)
  if n == 0 then return 1
  else return n * fact(n-1)
  end
end

要注意的是：这里local fact必须提前声明，否则递归的fact()只能去寻找全局的fact，全局的话不存在也不正确
问题：为什么方法2可以
方法2是一种语法糖，这等价于方法1

Tip：对于间接递归函数(由调用其它函数调回自己的递归方式)必须使用方法1，即明确前向声明
问题：为什么
如以下最简单的间接递归函数例子：

local f, g

function g()
    -- ...
    f()
    -- ...
end
function f()
    -- ...
    g()
    -- ...
end

在这里如果是前向声明那一定是可以的，但是如果使用语法糖，其实就变成了：

local g
function g()
    f()
end
local f
function f()
    g()
end

那么在g函数声明的时候f变量并没有赋值，当然是不可行的
所以：尽量不要使用语法糖形式，否则就需要仔细检查一下

闭包基础

闭包

闭包指的是：一个函数加上该函数所需访问的所有upvalue
闭包函数指的是：捕获到外部变量的那个函数，即内层函数
upvalue指的是：被捕获到的外部变量

-- Tip：比较函数比的是table的value
names = {"Peter", "Paul", "Mary"}
grades = {Mary = 10, Paul = 7, Peter = 8}
table.sort(names, function (n1, n2)
    return grades[n1] > grades[n2] -- 比较年纪
end)

-- 写成函数
function sortbygrade(names, grades)
    table.sort(names, function (n1, n2)
        return grades[n1] > grades[n2] -- 比较年纪
    end)
end

在这里有局部变量names grades，它们都被table.sort()内部函数所使用，由于grades在匿名函数中，它被称为非局部变量(non-local varibale)，即上值(upvalue)
问题：为什么grades是upvalue而name不是
以C#的说法的话，其实就是闭包存在于lambda表达式与局部函数(C#的)
在Lua的话虽然概念不同，但其实和C#完全一致，只有2种形式，即匿名函数与内部函数
那么names并不符合，当然不是upvalue

注意：同一个函数可以产生多个闭包

尾递归

function newCounter()
    local i = 0
    return function()
        i = i + 1
        return i
    end
end

c1 = newCounter()
print(c1()) -- 1
print(c1()) -- 2

c2 = newCounter()
print(c2()) -- 1
print(c1()) -- 3
print(c2()) -- 2

c1和c2的两次函数创建了不同的局部变量i，c1和c2当然是不同的闭包
此处使用到了尾调用---函数是另一个函数的最后一个动作
Lua支持尾调用消除，所以说：解释器会对尾调用优化，不会耗费任何栈空间
注意：尾递归必须是最后一个动作，如return g(x)+1由于加法操作，并不是最后一个动作，需注意
尾递归的一种应用：状态机

-- 每个房间都如此，那么虽然会在房间穿梭，
-- 但是由于发生的都是尾递归，所以尾递归消除的优化避免了栈的无限深入
function room1 ()
    local move = io.read()
    if move == "south" then 
        return room3()
    elseif move == "east" then 
        return room2()
    else
        print("invalid move")
        return room1() -- stay in the same room
    end
end

重定义函数

-- oldSin()即math.sin()被完全覆盖了，所以此时：
-- · math.sin()被覆盖为新的算法，我们只能调用到新的math.sin()
-- · 老的math.sin()已不复存在，我们只能通过新的math.sin()闭包调用oldSin访问到
-- 总的来说：是因为oldSin被包裹在了一个块中，那么出块后oldSin定义域一过就访问不了了，而math.sin是全局的，直接就被覆盖了
do
  local oldSin = math.sin
  local k = math.pi/180
  math.sin = function (x)
    return oldSin(x*k)
  end
end

安全的运行时环境

-- 在调用io.open()前先进行权限检查，通过才调用
do
    local oldOpen = io.open
    local access_OK = function(filename, mode)
        -- 检查访问权限
    end
    io.open = function(filename, mode)
        if access_OK(filename, mode) then
            return oldOpen(filename, mode)
        else
            return nil, "access denied"
        end
    end
end

函数式编程

function disk(cx, cy, r)
    return function(x, y)
        return (x - cx)^2 + (y - cy)^2 <= r^2
    end
end
  
function rect(left, right, bottom, up)
    return function(x, y)
        return left <= x and x <= right and bottom <= y and y <= up
    end
end
  
function complement(r)
    return function(x, y)
        return not r(x, y)
    end
end
  
function union(r1, r2)
    return function(x, y)
        return r1(x, y) or r2(x, y)
    end
end
  
function intersection(r1, r2)
    return function(x, y)
        return r1(x, y) and r2(x, y)
    end
end
  
function difference(r1, r2)
    return function(x, y)
        return r1(x, y) and not r2(x, y)
    end
end

function translate(r, dx, dy)
    return function(x, y)
        return r(x - dx, y - dy)
    end
end

-- 函数用于将区域函数r绘制为ASCII艺术
-- M是宽度，N是高度
function plot(r, M, N)
  io.write("P1\n", M, " ", N, "\n")  -- 写入PBM格式的文件头
  
  for i = 1, N do                    -- 对于每一行
    local y = (N - i*2)/N            -- 计算y坐标
    
    for j = 1, M do                  -- 对于每一列
      local x = (j*2 - M)/M          -- 计算x坐标
      
      io.write(r(x, y) and "1" or "0")  -- 如果点在区域内写1，否则写0
    end
    
    io.write("\n")                   -- 行结束换行
  end
end

迭代器与泛型for

迭代器

问题：迭代器是什么
迭代器无论是在Lua还是在C#中都是非常使用的一个内容，以下为基础用法：

//C#形式
foreach(var i in list)
{
    //...
}

-- lua形式
for k,v in pairs(list) do
  -- ...
end

以上都是泛型for形式
而迭代器的话是使用泛型for这种便捷方式的必须条件(在C#中的话就是IEnumerable接口(本质上是GetEnumerator函数))

原理探寻

在1_基础_变长参数中提到了ipairs其实不是什么语法糖，它就是一个函数
通过闭包，我们可以编写出和ipairs一样的函数：

-- 一种闭包的迭代器实现
function values(t)
    local i = 0
    return function() i = i + 1; return t[i] end
end

可以有：

-- while形式
t = {10,20,30}
iter = values(t)
while true do
    local element = iter()
    if element == nil then break end
    print(element)
end

-- 泛型for也可以
t = {10,20,30}
for element in values(t) do
    print(element)
end

可以看到这与C#中迭代器的本质是几乎相同的：

  1. 首先进行一次Reset()，也就是闭包中的i = 0
  2. 进行无限循环，每次都进行MoveNext()找到下一个值并且返回Current
  3. 找不到了说明遍历结束，退出即可

泛型for

泛型for其实就是一种语法糖，它会不断地调用迭代器函数获得输出并传入参数(in前的，如k v)，使用参数进行操作后再次调用迭代器函数，直到返回nil
具体来说，泛型for语法糖等价于：

-- 泛型for形式
for var_1,...,var_n in <explist> do
    <block>
end

-- 等价于
do
    local _f,_s,_var = <explist>
    while true do
        local var_1,...,var_n = _f(_s,_var)
        _var = var_1
        if _var == nil then break end
        <block>
    end
end

其中：

  • _f---迭代器函数
  • _s---恒定状态
  • _var---控制变量，指的是in前返回变量的第一个

也就是说，完整来说<explist>表达式需要输出迭代器函数/恒定状态/控制变量初始值以提供泛型for所需参数

在前面的values(t)例子中，我们的<explist>只返回了一个值，指的就是_f，_s和_var被补全为nil，这是通过闭包的方式代替了恒定状态与控制变量初始值
我们可以认为values(t)是一种简单迭代器
我们可以称完整的三参数迭代器为无状态迭代器---自身不保存任何状态，避免创建新闭包
最简单的例子就是Lua中的ipairs与pairs(源码为C)：

-- iapirs --
-- iter不再需要闭包，而是通过_s即a，_var即i完成
local function iter(a,i)
    i = i + 1
    local v = a[i]
    if v then
        return i,v
    end
end
function ipairs(a)
    return iter, a, 0 -- 参数3为_var的初始值
end

-- pairs --
function pairs(t)
    return next, t, nil -- 使用内置函数next()完成
end

例子：链表遍历

local function getnext(list, node)
    if not node then
        return list -- 第一次取头
    else
        return node.next
    end
end

function traverse(list)
    return getnext, list, nil
end

例子：key排序

对于一个table来说，我们想通过key进行排序，但是这无法做到，我们最简单可以通过转移到新表完成：

lines = {
    ["luaH_set"] = 10,
    ["luaH_get"] = 24,
    ["luaH_present"] = 48,
}
a = {}
for n in pairs(lines) do a[#a+1] = n end
table.sort(a) -- 排序(key并不会有所改变，还是1|2|3)
for k,n in ipairs(a) do print(k,n) end

注意：只有数组部分才能使用table.sort()，这也是成立的原因
另一种方法就是使用迭代器：

function pairsByKeys(t,f)
    local a = {}
    for n in pairs(t) do
        a[#a+1] = n
    end
    table.sort(a,f)
    local i = 0
    return function()
        i = i + 1
        return a[i], t[a[i]]
    end
end

for name, line in pairsByKeys(lines) do
    print(name, line)
end

与其说是另一种方法不如说是通过闭包的方式使其可以使用泛型for而已，做法还是一样的

老式迭代器

老式迭代器(因为没有泛型for)如下所示：　　即回调形式

function allwords(f)
    for line in io.lines() do
        for word in string,gmatch(line, "%w+") do
            f(word) -- 回调
        end
    end
end

allwords(print) -- 即找到单词后就print
--也可以有：
local count = 0
allwords(function(w)
    if w == "hello" then count = count + 1 end
end
print(count))
