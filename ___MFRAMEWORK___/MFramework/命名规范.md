<center><B><BBBG>命名规范</BBBG></B></center>

无论在任何地方编写代码，命名规范是很重要的一点，这<VT><B>确保了项目代码的一致性</B></VT>
具体使用哪种命名规范不重要但是一定要确立一种规范

<B>规范具体如下：</B>　　<VT>protected等价于private，internal等价于public</VT>

- <B>变量/常量</B>
  - <B>private</B>：使用前缀`_`+Camel
  - <B>public</B>：使用Pascal
  - <B>static</B>：
    - <B>private static</B>：使用前缀`s_`+Camel
    - <B>public static</B>：使用Pascal
  - <B>readonly</B>：使用前缀`_`+Camel
  - <B>static readonly</B>
    - <B>private static readonly</B>：使用前缀`s_`+Camel
    - <B>public static readonly</B>：使用Pascel
  - <B>const</B>：使用全大写+`_`分割
- <B>属性</B>：使用Pascal
- <B>枚举</B>：使用Pascal
- <B>事件</B>：使用前缀`On`+Pascal
- <B>类名/方法</B>：使用Pascal
    <BR>

    ``` csharp
    public class TClass 
    {
        public void Foo1() {/*...*/}
        public static void Foo2() {/*...*/}
    }
    public static class StringExtension
    {
        public static void Foo3(this string s) {/*...*/}
    }
    ```

稍微<B>总结</B>一下就是：
<B><VT>一般都是用Pascal，对于private的变量，会使用前缀+Camel的命名方式</VT></B>
